项目理解

一、项目是做什么的

这是一个 理财/金融管理 Web 系统，可以理解为一个小型“理财平台”：

  • 普通用户：登录后可以买零钱理财、定期理财、基金、工资理财，申请网贷，绑银行卡，看自己的理财记录和资金流水等。

  • 管理员：管理各类理财产品、审核网贷、管理用户和权限、发新闻、管银行信息等。

技术上是一个 Spring Boot + MyBatis + 页面模板 的 Java Web 项目，用 Shiro 做登录和权限控制。

二、整体架构（分层）

整体是常见的 “控制层 → 业务层 → 数据访问层” 结构，再加实体和公共组件：

浏览器请求

↓

Controller（控制层） ← 接收请求、跳转页面、返回 JSON

↓

Service（业务层） ← 写业务逻辑，调用 Mapper

↓

Mapper（数据访问层） ← MyBatis 接口，对应 SQL，访问数据库

↓

MySQL 数据库

同时还有：

  • entity：和数据库表对应的 Java 类（User、Admin、Loan、ChangeMoney 等）。

  • common：通用东西（统一返回格式 Msg、登录拦截、Shiro 配置、编码过滤器等）。

所以：架构 = 经典三层 + 实体 + 公共组件。

三、包结构大致分工
包名 作用（通俗说）
controller 处理每个“页面/接口”：谁点哪个链接、点哪个按钮，由哪个 Controller 方法管。例如登录、注册、我的理财、网贷、银行卡等。
service 业务逻辑：比如“买零钱理财”要查产品、创建用户理财记录、记流水，这些写在 Service 里。
service.impl 上面 Service 接口的具体实现类，里面会调用 Mapper。
mapper 和数据库打交道的接口，每个接口对应一张表或一类 SQL（查用户、查理财产品、插流水等）。
entity 数据库表对应的 Java 类，以及一些 Example（用于条件查询）。
common 公共配置和工具：统一返回格式、登录校验、Shiro 权限、错误页、编码过滤等。

启动类 FinanceApplication 里做了三件事：扫 Mapper、开事务、扫 Servlet/Filter，所以整个应用从这一处启动。

四、主要功能在代码里是怎么体现的
  1. 登录与权限

  • LoginController：用户/管理员登录校验、注册；用 Shiro 的 Subject.login(token) 做认证，成功后根据身份跳转 /user/index.html 或 /admin/index.html。

  • ShiroConfig：配置哪些路径必须登录、哪些放行（如 /、/login/、静态资源），以及 /user/ 需要 user 角色、/admin/** 需要 admin 角色。

  • LoginHandlerInterceptor：从 Session 里看有没有 loginUser 或 loginAdmin，没有就拦下来跳到登录页（当前在 MyMvcConfig 里被注释掉了，实际主要靠 Shiro 在做访问控制）。

  1. 用户端理财相关

  • 零钱理财：ChangeMoneyController → 展示零钱理财产品列表；用户“买入”时在 Service 里创建 UserChangeMoney 记录，并插入一条 FlowOfFunds 流水。

  • 我的理财汇总：MyFinanceController 的“我的理财”页面会查当前用户持有的：零钱理财、工资理财、定期理财、基金，并支持“撤销”某笔理财（把状态改为 3）。

  • 网贷：LoanController → 用户申请网贷（金额、期限、利率）、查看“我的借贷”列表；管理员侧有审核等（在别的 Controller 里）。

  • 其他类似：定期理财（TermFinancial）、基金（FundProduct）、工资理财（PayMoney）都是“产品表 + 用户持有表 + 流水”的模式，Controller 负责页面和接口，Service 负责业务，Mapper 负责数据库。

  1. 通用数据与返回格式

  • 列表分页用 PageHelper（如 LoanController 里 PageHelper.startPage + PageInfo）。

  • 前后端交互的 JSON 用 Msg 封装：Msg.success() / Msg.fail()，需要带数据时用 add(“key”, value)，前端根据 code（如 100 成功、200 失败）和 extend 里的数据做事。

  1. 视图与静态资源

  • 页面用 Thymeleaf 模板，放在 resources 下；MyMvcConfig 里把 / 和 /index.html 直接映射到登录页，/toregister.html 映射到注册页。

  • 静态资源（如 /asserts/、/js/ 等）在 Shiro 里做了放行，不登录也能访问。

五、数据流举例（买一笔零钱理财）

  1. 用户在前端选产品、点“买入”，请求发到：/user/buyChangeMoney（POST）。

  2. ChangeMoneyController.buyChangeMoney() 收到 changeMoneyId、userId。

  3. Controller 调 Service：查产品 ChangeMoney → 建一条 UserChangeMoney（用户持有）→ 再插一条 FlowOfFunds（资金流水）。

  4. Service 内部通过各个 Mapper（如 ChangeMoneyMapper、UserChangeMoneyMapper、FlowOfFundsMapper）读写数据库。

  5. 返回 Msg.success() 或 Msg.fail() 给前端，前端再刷新或跳转。

其它“买入”类功能（定期、基金、工资理财等）流程类似，只是表和业务字段不同。

六、小结（一句话版）

  • 架构：Spring Boot + 三层（Controller → Service → Mapper）+ 实体 + common（Msg、Shiro、拦截器等）。

  • 功能：用户端做各类理财购买、持有、撤销和网贷申请；管理员端做产品、借贷、用户与权限管理；登录与权限主要由 Shiro 和 Session 配合完成。

  • 代码分工：Controller 管“接到什么请求、返回什么”；Service 管“这件事怎么做”；Mapper 管“怎么读库写库”；entity 是表对应的模型；common 是全局共用配置和工具。
