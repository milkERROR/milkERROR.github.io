    随着移动支付的全面普及与家庭结构的小型化、核心化发展，传统手工记账与“一人管钱”模式已难以满足现代家庭对财务透明、共同决策与科学规划的迫切需求。针对现有个人记账软件缺乏协同机制、共同支出分摊困难、预算控制不足以及财务数据分散等问题，本文设计并实现了一套基于Spring Boot的家庭财务协同管理系统。

    系统采用B/S架构，以Java为开发语言，后端基于Spring Boot框架构建，集成MyBatis作为持久层、Apache Shiro作为安全认证框架，数据库选用MySQL。系统按照MVC三层架构进行设计，分为用户管理、家庭管理、账目管理、预算管理、财务目标管理和报表分析六个核心模块。针对家庭协同场景，系统设计了家长、财务官、普通成员三级角色权限体系，支持账单分摊（AA分摊、按比例分摊、按指定金额分摊）、隐私控制、计划外支出审批、预算超支预警、财务目标追踪及多维度报表分析等功能。

    经过功能测试与性能测试，系统各模块运行稳定，响应速度满足预期，能够有效支持家庭成员间的协同记账与财务规划。本系统为解决家庭财务管理中的协同与规划问题提供了一套可行的技术方案，具有一定的理论价值与实践意义。

    关键词：家庭财务协同管理；Spring Boot；MyBatis；权限控制；账单分摊

    With the widespread adoption of mobile payment and the trend toward smaller and more nuclear family structures, traditional manual bookkeeping and the “one-person-in-charge” model can no longer meet the urgent needs of modern families for financial transparency, joint decision-making, and scientific planning. To address the shortcomings of existing personal bookkeeping software—such as the lack of collaboration mechanisms, difficulties in sharing common expenses, insufficient budget control, and fragmented financial data—this paper designs and implements a family financial collaborative management system based on Spring Boot.

    The system adopts a B/S architecture, uses Java as the development language, and builds the backend on the Spring Boot framework, integrating MyBatis as the persistence layer and Apache Shiro as the security authentication framework, with MySQL as the database. Following the MVC three-tier architecture, the system consists of six core modules: user management, family management, account management, budget management, financial goal management, and report analysis. To address family collaboration scenarios, the system designs a three-level role permission system (parent, financial officer, and ordinary member), supporting bill sharing (equal sharing, proportional sharing, and designated-amount sharing), privacy control, out-of-plan expenditure approval, budget overrun alerts, financial goal tracking, and multi-dimensional report analysis.

    Functional and performance tests show that all modules operate stably and meet expected response times, effectively supporting collaborative bookkeeping and financial planning among family members. This system provides a feasible technical solution for addressing collaboration and planning challenges in family financial management, offering both theoretical value and practical significance.

关于权限

3.4 权限模型设计

本系统采用基于角色的访问控制（RBAC）模型，结合家庭财务协同场景的特点，预定义了“户主”、“成员”、“观察员”三种角色，并赋予不同的操作权限（见表3-2）。同时，为了兼顾部分家庭对权限的个性化需求，系统为户主提供了两个可选权限开关（“成员自删账单权限”、“成员导出报表权限”），户主可在“家庭设置”界面进行配置。该设计既保证了系统的易用性，又具备一定的灵活性。

评委问：“你的权限模型是固定的，还是可以让家长自定义？”

“系统默认提供了三种预定义角色，覆盖了绝大多数家庭的使用场景。同时，考虑到不同家庭对‘成员删除权限’和‘数据导出权限’可能有不同要求，我专门设计了两个可配置开关，让户主可以根据实际情况进行微调。这种设计避免了传统RBAC界面过于复杂的问题，既保证了安全性，又具备一定的灵活性。如果需要更细粒度的权限控制，可以作为后续扩展功能。”

评委追问：“为什么不做成完全动态的权限分配？”

“我在前期调研中发现，家庭用户往往不是专业管理员，过多的权限配置项会让用户感到困惑。因此我采用了‘约定优于配置’的思路，优先保证开箱即用。同时，我在代码层面使用了注解+拦截器的模式，如果未来需要扩展，可以很方便地升级为完整的动态权限模型。”

提供两个“可配置开关”作为家长细化授权的体现（既不复杂，又能展示灵活性）

在“家庭设置”页面，增加两个开关（仅户主可见）：

  • “允许成员删除自己的账单”（开关，默认关闭）
    如果打开，成员可以直接删除自己的账单；否则只能申请作废，由户主审批。
  • “成员是否可以导出家庭报表”（开关，默认关闭）
    防止成员将家庭财务数据导出外泄。

家长：最高权限角色，自动拥有所有权限
可以管理家庭成员（添加、修改、移除）
可以修改家庭信息
审批大额/计划外支出

财务官：专职财务管理角色
可以审批账单（与家长相同）
可以管理预算
不能管理家庭成员
不能修改家庭信息

普通成员：基础用户角色
✅ 可以录入自己的账单
✅ 查看账单（受隐私限制）
✅ 管理个人财务目标
✅ 查看报表
❌ 不能审批账单
❌ 不能管理预算
❌ 不能管理家庭成员
